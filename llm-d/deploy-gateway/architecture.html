📋 请求完整路径

1
Client → NodePort :32212
外部客户端发送 OpenAI 兼容 HTTP 请求，打到任意 worker 节点的 NodePort（当前 32212）
2
NodePort → svc/llm-d-inference-gateway → agentgateway proxy pod
Service 将请求转发到 agentgateway proxy pod（数据面），该 pod 由 agentgateway controller 自动创建
3
proxy 匹配 HTTPRoute
proxy 根据从 controller 获得的 xDS 配置，匹配 HTTPRoute/quickstart（path prefix /），确认转发目标为 InferencePool/quickstart
4
InferencePool → EPP 选路（gRPC :9002）
proxy 调用 EPP（quickstart-epp）的 gRPC 接口，EPP 根据各 vLLM pod 的 KV Cache 利用率、请求队列、前缀缓存命中率加权打分，返回最优 pod 的 IP 地址
5
proxy 直连 vLLM pod :8000
proxy 拿到 EPP 返回的 pod IP，直接将请求转发到该 vLLM pod 的 8000 端口，完成推理，结果原路返回
